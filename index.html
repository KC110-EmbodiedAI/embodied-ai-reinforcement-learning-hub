Learning Roadmap

四阶段学习路径

路线来自课程材料中的实践主线：用逐步增加复杂度的任务，帮助学生把强化学习概念转化为可调试、可部署、可解释的工程能力。

01

游戏仿真与奖励塑形

从可观察的游戏任务开始，理解状态、动作、奖励、失败现象和 MDP 建模。

  • 任务：记录失败现象并修改奖励设计
  • 产出：任务建模说明与实验记录
02

四足机器人仿真控制

进入连续控制问题，理解关节、动作空间、策略学习和仿真环境中的调参过程。

  • 任务：比较不同策略配置的训练表现
  • 产出：训练曲线、调参记录与行为分析
03

Sim2Real 与部署验证

关注仿真到真实系统的差异，建立风险清单、部署检查表和故障定位意识。

  • 任务：描述部署故障并提出排查路径
  • 产出：Sim2Real 风险清单与验证记录
04

证据化复盘与表达

从“结果展示”转向“证据说明”，用现象、问题、实验和改进解释自己的工程决策。

  • 任务：整理学习证据并完成阶段汇报
  • 产出：复盘报告、演示说明与自评表

Class Workflow

课堂任务流

每次实验建议围绕同一条证据链提交：先描述现象，再提出问题，随后给出建模、实验、部署和表达证据。

  1. 观察现象 记录策略失败、训练异常或部署不稳定。
  2. 定位问题 判断问题来自奖励、观测、动作、环境还是系统差异。
  3. 设计实验 提出可比较的实验设置和预期观察指标。
  4. 复盘表达 用证据解释改动原因、结果变化和下一步计划。
